# Changelog - v4

## [4.0.0](https://github.com/nishkohli96/rhf-mui-components/tree/v4.0.0)

**Released - 21 April, 2026**

### ⚙️ **Project & Build Setup**
- **Package Manager**: Migrated from **Yarn** → **pnpm** for faster, more efficient dependency management.
- Setup Script: Updated `setup.sh` to reflect the new pnpm workflow and installation process.

### 🧩 **Form Component Improvements**

#### General

- Developers can now manually trigger `rhfOnChange` when needed. For most components, this behavior is enabled via the `customOnChange` prop.
- Field errors are now handled internally. The `errorMessage` prop is **deprecated** across all components.
- Removed the `TrueOrFalse` type.
- Added `hideLabel` prop to all components to optionally hide the default `FormLabel`, enabling use of custom labels.
- Components like `RHFSelect`, which do not include a built-in label, can now have their label position controlled via the `allLabelsAboveFields` prop from `ConfigProvider`.
- Updated `onValueChange` to use an object-based argument instead of comma-separated values.
- Improved accessibility by applying `aria-*` props directly to the input element instead of the wrapper `div`.
- Added `customIds` prop to override default IDs for the field, label, and helper text elements.

#### Component Specific
In addition to the general improvements listed above, the following changes apply to specific components:

##### NumberInput 🧮
- Added `onlyIntegers` prop - allows only integer values (positive, negative, or zero).
- Added `nonNegative` prop - restricts input to non-negative values.
- Added `maxDecimalPlaces` prop — limits the number of digits allowed after the decimal point.
- Added `stepAmount` prop — controls the increment/decrement value when using arrow keys or input steppers.


### 🔍 Autocomplete Inputs
The fixes apply to both `RHFAutocomplete` and `RHFMultiAutocomplete`:
- Fixed **generic type handling** to improve type safety and flexibility for string or object-based options
- Optimized rendering logic and memoization for smoother performance and cleaner re-renders
- Add `renderOption` prop in `RHFAutocomplete` & `renderOptionLabel` in `RHFMultiAutocomplete`

### ✅ Select & Choice Inputs
- Added `getOptionDisabled` to disable specific options dynamically:
  - `RHFCheckboxGroup`
  - `RHFRadioGroup`
  - `RHFSelect`
  - `RHFNativeSelect`
- Added `renderOption` prop to enable full control over option rendering:
  - `RHFSelect`
  - `RHFNativeSelect`
	- `RHFCheckboxGroup`
  - `RHFRadioGroup`
- Fixed **generic type handling** to improve type safety and flexibility for string, number, or object-based options:
  - `RHFSelect`
  - `RHFNativeSelect`
	- `RHFCheckboxGroup`
  - `RHFRadioGroup`

### 🗓️ MUI Pickers Enhancements
- Added **Desktop**, **Mobile**, and **Static** DateTimePicker components in the `mui-pickers` module for consistent and device-optimized date/time selection.
- Trigger `rhfOnBlur` event `onAccept`


### 📁 File Upload
- Added `uploadedFiles` prop to `RHFFileUploader` for externally controlled file lists.
- Updated `validateFileList` to accept both FileList and File[].
- Introduced **optional drag-and-drop** functionality for a smoother upload experience.

### ☑️ Accessibility Enhancements
- Added `aria-checked` and `aria-disabled` attributes to all `checkbox` and `radio` buttons for better screen reader compatibility.

### 🏷️ RHFTagsInput Optimization
- Memoized internal functions and derived values to reduce unnecessary re-renders and improve performance.

### RHFCountrySelect
- Remove `displayFlagOnSelect` prop

### RHFRichTextEditor
- Add `ref` prop to control `CKEditor`

### 📱 Phone Input Optimization
- Memoized `startAdornment` containing the **country list options** in `RHFPhoneInput` for performance improvements.

### 🏷️ Form Labels
- Added a new `hideLabel` prop to all relevant components, allowing you to hide both the `FormLabel` text and the default label rendered by the component itself.


### Enhancements ✨
- Fix docs links to correctly point to their respective versioned documentation.
- Fix example links in older versioned docs to point to the latest examples website.

### Update Packages
- Docusaurus from `3.7.0` to `3.9.2`
- Upgrade Next.js app from `v15` to `v16`
- Upgraded node version from `22` to `24`

### ⚠️ Error Messages
- Updated error message text for `generateLabelValueErrMsg` and `generateDateAdapterErrMsg` to improve clarity and consistency.
