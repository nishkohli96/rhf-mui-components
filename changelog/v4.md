# Changelog - v4

## [4.0.0](https://github.com/nishkohli96/rhf-mui-components/tree/v4.0.0)

**Released — June 24, 2026**

### ⚙️ Project & Tooling
- Migrated from **Yarn** to **pnpm** for faster and more efficient dependency management.
- Change `moduleResolution` from `node` to `bundler`
- Updated `setup.sh` to support the pnpm-based workflow.
- Standardized the Node.js version to **v24.14.0** via `.nvmrc`.
- Upgraded:
  - **Docusaurus**: `3.7.0` → `3.10.0`
  - **Next.js**: `15` → `16`
  - **Node.js**: `22` → `24`

### 🧩 Core Library Improvements
- Added support for manually triggering `rhfOnChange` through the new `customOnChange` prop across supported components.
- Updated all `onValueChange` callbacks to use a single object-based argument instead of positional parameters.
- Components now respect the form-level `disabled` option passed through React Hook Form's `useForm` hook.
- Added `hideLabel` to all components to optionally hide the default `FormLabel`.
- Added `customIds` to override generated field, label, helper text, and error IDs.
- Improved accessibility by applying `aria-*` attributes directly to input elements instead of wrapper containers.
- Added external `ref` forwarding support across all components. Component refs are now merged with React Hook Form’s internal field refs to preserve RHF registration and focus behavior.
- Components without built-in labels (such as `RHFSelect`) now respect the global `allLabelsAboveFields` configuration.
- Deprecated the `errorMessage` prop. Validation errors are now handled internally.
- Removed the unused `TrueOrFalse` type.
- Added comprehensive JSDoc documentation for component-specific props.
- Updated version-specific documentation and prop references.
- Switched position of `RHFPasswordInput` and `RHFNumberInput` in docs.
- Narrowed password and number input event types from `HTMLInputElement | HTMLTextAreaElement` to `HTMLInputElement`.

### 🧩 Component-Specific Enhancements
In addition to the core library improvements, several components received new features, customization options, and behavioral improvements.

#### 🧮 RHFNumberInput
Added new input validation and stepping controls:
- `onlyIntegers` — Restricts input to integer values only.
- `nonNegative` — Restricts input to non-negative values.
- `maxDecimalPlaces` — Limits the number of digits allowed after the decimal point.
- `stepAmount` — Controls the increment/decrement amount for keyboard and stepper interactions.

#### 🏷️ RHFTagsInput
Added advanced tag management capabilities:
- `onTagAdd`
- `onTagDelete`
- `onTagPaste`

Additional configuration options:
- `delimiter` — Defines the character used to separate tags.
- `maxTags` — Limits the number of tags that can be added.

#### 📁 RHFFileUploader

##### Added
- Added **drag-and-drop** file upload support (enabled by default and can be disabled using `disableDragAndDrop`).
- Added `existingFiles` for displaying and managing pre-existing server-side files.
- Added `dropZoneProps` for customizing the drag-and-drop area.
- Added `renderExistingFileItem` for custom rendering of existing files.
- Added `existingFileListProps` and `uploadedFileListProps` for customizing file list containers.
- Added `customOnChange` for full control over file validation and form state updates.
- Added support for removing uploaded files via the `renderFileItem` callback.

##### Changed
- `validateFileList` now supports both `FileList` and `File[]`.
- Improved required field validation when `existingFiles` are provided.
- Improved upload count validation by including `existingFiles` in `maxFiles` calculations.
- `onUploadError` now returns detailed file-specific validation errors instead of separate error and file arrays.
- Existing files and newly uploaded files are rendered only when `renderExistingFileItem` and `renderFileItem` are provided, respectively.

##### Removed
- `errorMessage` prop as it is now handled internally within the component.

#### 🔘 Select, NativeSelect, CheckboxGroup & RadioGroup
Added support for advanced option customization:
- `getOptionDisabled` — Dynamically disable specific options.
- `renderOptionLabel` — Fully customize option rendering.

#### 🔍 Autocomplete & MultiAutoComplete
##### Added
- Added support for the `freeSolo` prop in `RHFAutocomplete` and `RHFMultiAutocomplete`.
- Added `renderOptionLabel` prop to `RHFMultiAutocomplete` & `RHFMultiAutocompleteObject` for customizing checkbox option labels.

##### Changed
- `autoHighlight` is now configurable via props and defaults to `true`.
- In `RHFMultiAutocomplete`, the "**Select All**" option is automatically hidden when `freeSolo` is enabled.

#### 🌍 RHFCountrySelect
##### Added
- `renderOptionLabel` for customizing the content rendered within each country option.

##### Changed
-  When `valueKey` is provided, selected value(s) are stored using the specified country property.
- When `valueKey` is omitted, selected value(s) are stored as complete country objects.

##### Removed
- Removed the `displayFlagOnSelect` prop.

#### 🗓️ MUI Pickers
- Added Desktop, Mobile, and Static DateTime Picker variants.
- `rhfOnBlur` is now triggered from `onAccept` to improve validation behavior.

#### 🎨 RHFColorPicker
##### Removed
- `value` prop is no longer needed as the selected color value is now derived automatically from React Hook Form state.

#### ✍️ RHFRichTextEditor
- Added support for forwarding refs to the underlying CKEditor instance.

#### 📱 RHFPhoneInput

##### Added
- Added country search support within the country selection menu, allowing users to quickly find and select a country.
- Added `searchCountryProps` for customizing the country search field and country menu item rendering.
- `disableDropdown` prop to disables the dropdown to select country.

##### Changed
- The field value is now returned as an `RHFPhoneInputValue` object instead of a plain `string`, providing access to additional metadata such as the selected country, dial code, and national phone number.

##### Removed
- `value` prop is no longer needed as it is now derived automatically from React Hook Form state.
- `errorMessage` prop as it is now handled internally within the component.
- Removed `hideDropdown` from `PhoneInputProps` in favor of the `disableDropDown` prop on `RHFPhoneInput`.

### ♿ Accessibility
- Added `aria-checked` and `aria-disabled` attributes to all checkbox and radio controls.
- Improved overall accessibility through more accurate element-level ARIA attributes.

### 📚 Documentation
- Fixed versioned documentation links.
- Fixed example links in older documentation versions to point to the latest examples site.
- Expanded component prop documentation and version-specific API references.

### ⚠️ Error Messages
Improved the clarity and consistency of:
- `generateLabelValueErrMsg`
- `generateDateAdapterErrMsg`
