# Changelog - v3

## [3.2.0](https://github.com/nishkohli96/rhf-mui-components/tree/v3.2.0)

**Released - 28 December, 2025**

### Enhancements üîß‚ú®
- Fixed **generic type handling** to improve type safety and flexibility for string or object-based options for the following components:
  - `RHFAutocomplete`
  - `RHFCheckboxGroup`
  - `RHFMultiAutocomplete`
  - `RHFNativeSelect`
  - `RHFRadioGroup`
  - `RHFSelect`
- Add typing for `labelKey` and `valueKey` props for components using object based options.

### Bug Fixes üêû
- Changed default value of the `autoComplete` prop from `fieldName` ‚Üí `"off"` to prevent unwanted browser autocomplete behavior across all input and select fields.
- `RHFCountrySelect`: Handle spread of `InputProps` and `inputProps`for MUI **v5**
- `RHFNativeSelect`
    - Cast value from string to number if options is an array of numbers.
    - Show `defaultOptionText` or `placeholder` in defaultOption text, else show empty string.
- `RHFSelect`: Make the `placeholder` text blurred when no option is selected.
