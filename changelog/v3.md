## [3.3.0](https://github.com/nishkohli96/rhf-mui-components/tree/v3.3.0)

**Released - 26 March, 2026**

### New Components 🔥
- [RHFAutocompleteObject](../apps/rhf-mui-docs/docs/components/mui/RHFAutocompleteObject.mdx)
    - Supports object-based options
    - Returns full selected object(s) instead of primitive values
- [RHFMultiAutocompleteObject](../apps/rhf-mui-docs/docs/components/mui/RHFMultiAutocompleteObject.mdx)
    - Multi-select autocomplete with object values
    - Built-in Select All support with indeterminate state handling

### Enhancements ✨
- Add **RHF-MUI** logo for docs and demo website 
- Improved accessibility:
    - Auto-generate `id`, `labelId`, `helperTextId`, and `errorId` from fieldName
    - Properly link `FormLabel` and `FormHelperText`
- Forwarded `field.ref` to underlying MUI input components for better form integration
- Exported `selectAllOptionValue` of `RHFMultiAutocomplete` and `RHFMultiAutocompleteObject` from **mui** module 
    - Enables easier comparison inside onValueChange
- Updated `RHFMultiAutocomplete`:
    - Improved `onChange` logic for better consistency and edge-case handling
- Added validation schema examples for date-time pickers
- Add "Reset Button" in all example forms

### Bug Fixes 🐞
- **Field IDs**
  - Normalized HTML `id` attributes using `fieldNameToId`
  - Example: `user.0.accountNumber` → `user-0-accountNumber`
- **`FormHelperText`**
  - Hidden when both `helperText` and `errorMessage` are undefined
- **`RHFPasswordField`**
  - Spread `inputProps` along with `endAdornment`
  - Removed unsupported `multiline` prop
  - Fixed event typing (`ChangeEvent<HTMLInputElement>`)
- **`RHFNumberInput`**
  - Removed unsupported `multiline` prop
  - Fixed event typing
- **`RHFTagsInput`**
  - Fixed missing error state reflection in `TextField`
  - Prevented duplicate tag entries
  - Ensured compatibility with MUI v5 (`InputProps` spread)
- **`RHFSelect` & `RHFNativeSelect`**
  - Default option is now disabled only when the field is required
- **`RHFSelect`**
  - Fixed label overlapping border when focused with no value
- **`RHFCountrySelect`**
  - Set value to `null` when cleared in single-select mode
- **`RHFCheckboxGroup`**
  - Fixed blur behavior: blur fires only when focus moves outside the group, avoiding redundant blur events

### Optimizations 🔧
- **validateArray**: Skipped validation for option based components in `production` environment to avoid expensive runtime checks
- Updated `tsdown` from `0.20.3` to `0.21.4`

## [3.2.0](https://github.com/nishkohli96/rhf-mui-components/tree/v3.2.0)

**Released - 23 February, 2026**

### Enhancements ✨
- Fixed **generic type handling** to improve type safety and flexibility for string or object-based options, as well as `labelKey` and `valueKey` props for the following components:
  - `RHFAutocomplete`
  - `RHFCheckboxGroup`
  - `RHFMultiAutocomplete`
  - `RHFNativeSelect`
  - `RHFRadioGroup`
  - `RHFSelect`
- Add `hideSelectAllOption` prop in `RHFMultiAutocomplete` to hide the "Select All" option.
- Add more examples for cases in forms.

### Bug Fixes 🐞
- Changed default value of the `autoComplete` prop from `fieldName` → `"off"` to prevent unwanted browser autocomplete behavior across all input and select fields.
- `RHFCountrySelect`: Handle spread of `InputProps` and `inputProps`for MUI **v5**
- `RHFNativeSelect`
    - Cast value from string to number if options is an array of numbers.
    - Show `defaultOptionText` or `placeholder` in defaultOption text, else show empty string.
- `RHFSelect`: Make the `placeholder` text blurred when no option is selected.
- `RHFMultiAutocomplete`: Improved selection logic and fixed checkbox sync issues with `defaultValues`.
