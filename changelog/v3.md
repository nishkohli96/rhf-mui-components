## [3.4.0](https://github.com/nishkohli96/rhf-mui-components/tree/v3.4.0)

**Released - 29 June, 2026**

### ⚙️ Project & Tooling
- Update `turbo` from `2.5.2` to `2.10.0`.
- Add `prepare-package-json.mjs` script to generate `package.json` for **rhf-mui** build.
- Update `turbo.json` to correct `build`, `run` and `lint` scripts.
- Reorganized project scripts under the root `scripts` directory for better maintainability.
- Updated GitHub Actions to use `scripts/setup.sh` as the single setup entry point for app initialization.
- Add `vercel-ignore-build` script to prevent deploys for `v3` branch or tags on **Vercel**.

### ✨ Enhancements
- Update [Netlify Examples App](https://rhf-mui-components-examples.netlify.app/) production branch as `version-3`
- Added comprehensive JSDoc documentation for props across all form components.
- Added a "**Disable all fields**" checkbox to each demo form.
- Updated the package `README.md` to reference the new Vercel-hosted documentation.
- Update `actions/checkout` from `v6` to `v7`
- Updated package `tsconfig.json`.
- Improved autocomplete generics by threading `Multiple` and `DisableClearable` through autocomplete variants. `TrueOrFalse` type is now no longer required.
- Refined ARIA handling across components by removing redundant invalid states and ensuring accessibility props are applied to the correct input elements.

### 🐞 Bug Fixes
This update includes the following fixes in **v4**:
- Components now correctly respect the `disabled` option passed through React Hook Form's `useForm` configuration.
- Passed the `disabled` prop through to the underlying `FormControl` or `FormControlLabel` components.
- Refactored component structure by setting `Controller` the parent of `FormControl`, `FormLabel`, and `FormHelperText`.
- Fixed `RHFFileUploader` uploads not registering after file selection, caused in `v3.3.0`.
- Updated `RHFFileUploader`'s `renderFileItem` callback to include a third argument for removing the uploaded file.
- `RHFColorPicker`: Fixed an issue where `rhfOnChange` was not triggered when the `Hue` component value changed.
- Fixed demo website theme flicker where light mode briefly switched to dark on load.
- Components without built-in labels (such as `RHFSelect`) now respect the global `allLabelsAboveFields` configuration.
- Removed `validateArray` from option-based components.

## [3.3.0](https://github.com/nishkohli96/rhf-mui-components/tree/v3.3.0)

**Released - 26 March, 2026**

### New Components 🔥
- [RHFAutocompleteObject](../apps/rhf-mui-docs/docs/components/mui/RHFAutocompleteObject.mdx)
    - Supports object-based options
    - Returns full selected object(s) instead of primitive values
- [RHFMultiAutocompleteObject](../apps/rhf-mui-docs/docs/components/mui/RHFMultiAutocompleteObject.mdx)
    - Multi-select autocomplete with object values
    - Built-in Select All support with indeterminate state handling

### Enhancements ✨
- Add **RHF-MUI** logo for docs and demo website 
- Improved accessibility:
    - Auto-generate `id`, `labelId`, `helperTextId`, and `errorId` from fieldName
    - Properly link `FormLabel` and `FormHelperText`
- Forwarded `field.ref` to underlying MUI input components for better form integration
- Exported `selectAllOptionValue` of `RHFMultiAutocomplete` and `RHFMultiAutocompleteObject` from **mui** module 
    - Enables easier comparison inside onValueChange
- Updated `RHFMultiAutocomplete`:
    - Improved `onChange` logic for better consistency and edge-case handling
- Added validation schema examples for date-time pickers
- Add "Reset Button" in all example forms

### Bug Fixes 🐞
- **Field IDs**
  - Normalized HTML `id` attributes using `fieldNameToId`
  - Example: `user.0.accountNumber` → `user-0-accountNumber`
- **`FormHelperText`**
  - Hidden when both `helperText` and `errorMessage` are undefined
- **`RHFPasswordField`**
  - Spread `inputProps` along with `endAdornment`
  - Removed unsupported `multiline` prop
  - Fixed event typing (`ChangeEvent<HTMLInputElement>`)
- **`RHFNumberInput`**
  - Removed unsupported `multiline` prop
  - Fixed event typing
- **`RHFTagsInput`**
  - Fixed missing error state reflection in `TextField`
  - Prevented duplicate tag entries
  - Ensured compatibility with MUI v5 (`InputProps` spread)
- **`RHFSelect` & `RHFNativeSelect`**
  - Default option is now disabled only when the field is required
- **`RHFSelect`**
  - Fixed label overlapping border when focused with no value
- **`RHFCountrySelect`**
  - Set value to `null` when cleared in single-select mode
- **`RHFCheckboxGroup`**
  - Fixed blur behavior: blur fires only when focus moves outside the group, avoiding redundant blur events

### Optimizations 🔧
- **validateArray**: Skipped validation for option based components in `production` environment to avoid expensive runtime checks
- Updated `tsdown` from `0.20.3` to `0.21.4`
