# Migrate to v4

- **MUI v5 is no longer supported**. Please upgrade **Material UI** to `v6` or `v7`, or continue using the latest published `v3` version of this package.
- The `errorMessage` prop has been deprecated across all components and **will be removed** in the next major upgrade. Field errors are now handled internally.

## Global Breaking Changes
- `onValueChange` **signature updated**

  The function now accepts an object instead of positional arguments and is of the form:

	```ts
	({ newValue, event, ...otherArgs })
  ```
  where,
	- `newValue`: The latest value of the field after user interaction
	- `event`: The event that triggered the value update
	- `otherArgs`: Additional arguments specific to each component's implementation.

  This change ensures consistency with the `customOnChange` prop across all components.

- **Date & Time Picker imports updated**

  With additional variants from MUI Pickers (`Desktop`, `Mobile` and `Static`), imports have been simplified and made more explicit.

  **Use:**
  ```js
  import {
    RHFDatePicker,
    RHFMobileDatePicker
  } from '@nish1896/rhf-mui-components/mui-pickers/date';
  ```

  **Instead of:**

  ```js
  import RHFDatePicker from '@nish1896/rhf-mui-components/mui-pickers/date';
  ```

## Component Specific Changes
The following components include new features, behavioral changes, and breaking API updates that may require code changes when migrating from `v3` to `v4`.

### 📁 RHFFileUploader {#rhffileuploader}

#### Added
- **drag-and-drop** uploads (enabled by default).
- Support for pre-existing server-side files via `existingFiles`.
- `dropZoneProps` for customizing the drag-and-drop area.
- `customOnChange` for custom validation and manual control of form state updates.
- Support for removing uploaded files via the `removeFile` helper provided to `renderFileItem`.

#### Removed Props
The following props have been removed:
- `errorMessage` — Validation errors are now derived automatically from React Hook Form state.
- `hideFileList` — File rendering is now fully controlled via `renderFileItem` and `renderExistingFileItem`.
- `showFileSize` — File size rendering must now be handled within custom file renderers.

#### Breaking Changes
- `onValueChange` now receives a single object argument instead of positional parameters.
- `renderFileItem` now receives an object containing `file`, `index`, and `removeFile`.
- `onUploadError` now returns file-specific validation details instead of separate error and file arrays.

#### File Rendering
Default rendering of uploaded and pre-existing files has been removed.
To display files, you must now provide:
- `renderFileItem` — Renders newly uploaded files.
- `renderExistingFileItem` — Renders files provided through `existingFiles`.

This change provides complete control over the appearance and behavior of file lists.

:::note
Please refer to the updated [RHFFileUploader](../components/mui/RHFFileUploader.mdx) documentation for complete usage examples and migration details.
:::

### 🌍 RHFCountrySelect
##### Changed
-  When `valueKey` is provided, selected value(s) are stored using the specified country property.
- When `valueKey` is omitted, selected value(s) are stored as complete country objects.

:::warning
If you previously relied on the default `valueKey` behavior (ie. `iso`), review your form values and submission payloads after upgrading.
:::

### 🎨 RHFColorPicker
#### Removed
- `value` prop is no longer needed as the selected color value is now derived automatically from React Hook Form state.

#### 📱 RHFPhoneInput

##### Changed
- The field value is now returned as an `RHFPhoneInputValue` object instead of a plain `string`, providing access to additional metadata such as the selected country, dial code, and national phone number.

##### Removed
- `value` prop is no longer needed as it is now derived automatically from React Hook Form state.
- `errorMessage` prop as it is now handled internally within the component.
- Removed `hideDropdown` from `PhoneInputProps` in favor of the `disableDropDown` prop on `RHFPhoneInput`.
  ```diff
    <RHFPhoneInput
      fieldName="phoneNumber"
      control={control}
    />
  ```
