# Migrate to v4

- **MUI v5 is no longer supported**. Please upgrade **Material UI** to `v6` or `v7`, or continue using the latest published `v3` version of this package.
- The `errorMessage` prop has been deprecated across all components and **will be removed** in the next major upgrade. Field errors are now handled internally.

## Global Breaking Changes
- `onValueChange` **signature updated**

  The function now accepts an object instead of positional arguments and is of the form:

  ```ts
  ({ newValue, event, ...otherArgs })
  ```
  where,
  - `newValue`: The latest value of the field after user interaction
  - `event`: The event that triggered the value update
  - `otherArgs`: Additional arguments specific to each component's implementation.

  This change ensures consistency with the `customOnChange` prop across all components.

- `customOnChange` **signature updated**

  Components that support `customOnChange` now receive an object argument instead of positional arguments.

  ```ts
  ({ rhfOnChange, newValue, event, ...otherArgs })
  ```

  :::warning
  When using `customOnChange`, call `rhfOnChange(newValue)` manually to update React Hook Form state.
  `onValueChange` is not called when `customOnChange` is provided.
  :::

- **React Hook Form disabled state is now respected**

  Components now read the `disabled` value provided by React Hook Form's `Controller` field state. This means a form-level disabled state passed through `useForm({ disabled: true })` disables supported components automatically.

  :::warning
  Disabled fields follow React Hook Form and native form behavior. Depending on how you submit or serialize values, disabled fields may be omitted from the submitted payload.
  :::

- **Date & Time Picker imports updated**

  With additional variants from MUI Pickers (`Desktop`, `Mobile` and `Static`), imports have been simplified and made more explicit.

  **Use:**
  ```js
  import {
    RHFDatePicker,
    RHFMobileDatePicker
  } from '@nish1896/rhf-mui-components/mui-pickers/date';
  ```

  **Instead of:**

  ```js
  import RHFDatePicker from '@nish1896/rhf-mui-components/mui-pickers/date';
  ```

## Component Specific Changes
The following components include new features, behavioral changes, and breaking API updates that may require code changes when migrating from `v3` to `v4`.

### Autocompletes
- `RHFAutocomplete` and `RHFMultiAutocomplete` now support the `freeSolo` prop, allowing users to enter values that are not present in the provided `options`.
- If enabling `freeSolo`, review any custom `renderValue`, `renderTags`, or `renderOptionLabel` implementations to ensure they correctly handle both option objects and user-entered string values (for example, using `typeof value === 'string'`).

### Þ RHFFileUploader {#rhffileuploader}

#### Added
- **drag-and-drop** uploads (enabled by default, can be disabled using ``disableDragAndDrop` prop).
- Support for pre-existing server-side files via `existingFiles`.
- Support for removing uploaded files via the `removeFile` helper provided to `renderFileItem`.

#### Removed Props
The following props have been removed:
- `errorMessage` ã Validation errors are now derived automatically from React Hook Form state.
- `hideFileList` ã File rendering is now fully controlled via `renderFileItem` and `renderExistingFileItem`.
- `showFileSize` ã File size rendering must now be handled within custom file renderers.

#### Breaking Changes
- `onValueChange` now receives a single object argument instead of positional parameters.
- `renderFileItem` now receives an object containing `file`, `index`, and `removeFile`.
- `onUploadError` now returns file-specific validation details as an array of `{ file, errors }` objects instead of separate error and rejected file arrays.
- When `multiple` is enabled, newly uploaded files are appended to the current RHF field value instead of replacing it.
- `maxFiles` validation now also counts `existingFiles`.
- Built-in required validation treats `existingFiles.length > 0` as satisfying the required rule.
- Method signature of `validateFileList` has also been updated.

```diff
- validateFileList(fileList, '*', 5 * 1024 * 1024, 3)
+ validateFileList(fileList, { accept: '*', maxSize: 5 * 1024 * 1024, maxFiles: 3 });
```

#### Callback Migration

```diff
  <RHFFileUploader
    fieldName="documents"
    control={control}
    multiple
-   onValueChange={(acceptedFiles, event) => {
-     console.log(acceptedFiles, event);
-   }}
+   onValueChange={({ newValue, event }) => {
+     console.log(newValue, event);
+   }}

-   onUploadError={(errors, rejectedFiles) => {
-     console.log(errors, rejectedFiles);
-   }}
+   onUploadError={fileErrors => {
+     fileErrors.forEach(({ file, errors }) => {
+       console.log(file.name, errors);
+     });
+   }}

-   renderFileItem={(file, index) => (
-     <UploadedFile key={file.name} file={file} index={index} />
-   )}
+   renderFileItem={({ file, index, removeFile }) => (
+     <UploadedFile
+       file={file}
+       index={index}
+       onRemove={removeFile}
+     />
+   )}

-   hideFileList
-   showFileSize
-   errorMessage={errors?.documents?.message}
  />
```

:::note
Please refer to the updated [RHFFileUploader](../components/mui/RHFFileUploader.mdx) documentation for complete usage examples and migration details.
:::

### Þ RHFCountrySelect
#### Changed
- When `valueKey` is provided, selected value(s) are stored using the specified country property.
- When `valueKey` is omitted, selected value(s) are stored as complete country objects.

:::warning
If you previously relied on the default `valueKey` behavior (i.e. `iso`), review your form values and submission payloads after upgrading.
:::

### Þ´ RHFColorPicker
#### Removed
- `value` prop is no longer needed as the selected color value is now derived automatically from React Hook Form state.

### ÞÝ RHFPhoneInput {#rhfphoneinput}

#### Added
- The country dropdown now includes an inline search field by default.

#### Changed
- The field value is now stored as an `RHFPhoneInputValue` object instead of a plain `string`, providing access to the selected country, dial code, and national phone number.

  ```diff
  import type { RHFPhoneInputValue } from '@nish1896/rhf-mui-components/misc/phone-input';

  type FormValues = {
  - phoneNumber: string;
  + phoneNumber: RHFPhoneInputValue;
  };
  ```

  ```diff
    const { control } = useForm<FormValues>({
      defaultValues: {
  -    phoneNumber: '+918885511000'
  +    phoneNumber: {
  +       phone: '+918885511000',
  +       country: 'in',
  +       dialCode: '91',
  +       phoneNo: '8885511000'
  +     }
      }
    });
  ```

  ##### Validation Changes
  Update string-based validations to read from the structured value.

  ```diff
    <RHFPhoneInput
      fieldName="phoneNumber"
      control={control}
      registerOptions={{
        required: 'Phone number is required',
  -     validate: value =>
  -       value.length >= 6 || 'Enter a valid phone number'
  +     validate: value =>
  +       value.phoneNo.length >= 6 || 'Enter a valid phone number'
      }}
    />
  ```

  :::warning
  If your API still expects a string, submit `phoneNumber.phone`. Use `country`, `dialCode`, and `phoneNo` when your backend needs the selected country or the national number separately.
  :::
- `onValueChange` now receives `{ newValue, phoneData }`.
  ```diff
    <RHFPhoneInput
      fieldName="phoneNumber"
      control={control}
  -   onValueChange={phoneData => {
  -     console.log(phoneData.phone);
  -   }}
  +   onValueChange={({ newValue, phoneData }) => {
  +     console.log(newValue.phone, phoneData.country);
  +   }}
    />
  ```

#### Removed
- `value` prop is no longer needed as it is now derived automatically from React Hook Form state.
- `errorMessage` prop as field errors are now derived automatically from React Hook Form state.
- `phoneInputProps.hideDropdown` has been removed. Use `phoneInputProps.forceDialCode` when you want to lock the selected country/dial code.

```diff
  <RHFPhoneInput
    fieldName="phoneNumber"
    control={control}
    phoneInputProps={{
      defaultCountry: 'in',
-     hideDropdown: true
+     forceDialCode: true
    }}
-   value={watch('phoneNumber')}
-   errorMessage={errors?.phoneNumber?.message}
  />
```

:::note
Please refer to the updated [RHFPhoneInput](../components/misc/RHFPhoneInput.mdx) documentation for complete usage and examples.
:::
